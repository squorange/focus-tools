# Bug Tracker

Persistent bugs that span multiple debugging sessions. Purpose: preserve context, track attempted fixes, avoid duplicate efforts.

**Usage:** Add bugs here when they've resisted 2+ fix attempts, or when you need to pause debugging and preserve context for a future session.

---

## Template

```markdown
## [Status] Bug Title

**First observed:** YYYY-MM-DD
**Last updated:** YYYY-MM-DD
**Severity:** P0 (blocking) | P1 (major) | P2 (minor) | P3 (cosmetic)

### Reproduction
1. Step one
2. Step two
3. Expected: X
4. Actual: Y

### Root Cause Hypothesis
What we think is happening and why.

### Relevant Code
- `path/to/file.tsx:123` - description
- `path/to/other.tsx:456` - description

### Attempted Fixes

| Date | Attempt | Outcome | Why it failed |
|------|---------|---------|---------------|
| YYYY-MM-DD | Description of change | Did not fix | Explanation |

### Current Observations
Notes from debugging, console output, dev tools findings, etc.

### Next Steps
What to try next, hypotheses to test.

### Resolution
_(Fill in when resolved)_
- **Fixed:** YYYY-MM-DD
- **Solution:** What actually worked
- **Learning:** Why previous attempts failed
```
